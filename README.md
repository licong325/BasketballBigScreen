# 篮球计分大屏系统

一个纯静态 HTML 的篮球比赛计分大屏系统，适合球馆比赛现场在同一台电脑上完成赛事配置、记分控制和外接大屏展示。

项目不需要服务器、不需要数据库、不需要安装依赖，可以直接部署到 GitHub Pages。

## 页面说明

- `login.html`：登录页
  - 默认入口页面
  - 使用前端预置账号登录
  - 当前默认账号：`demo`
  - 当前默认密码：`123456`

- `index.html`：赛事配置页
  - 配置比赛标题
  - 配置主队、客队名称
  - 配置主客队球衣颜色
  - 录入主客队球员
  - 配置是否显示比赛时间、24 秒
  - 配置球员排序方式
  - 打开记分板和比分大屏

- `scoreCtrl.html`：记分控制台
  - 控制比赛时间和 24 秒
  - 调整每节比赛时长
  - 球员加 1/2/3 分
  - 记录球员犯规
  - 记录球队暂停
  - 设置球权
  - 勾选上场球员
  - 支持主控锁，避免同一浏览器多开控制台同时操作
  - 支持按球员撤回最近一次加分或犯规

- `bigScreen.html`：比分大屏
  - 展示比赛标题
  - 展示主客队比分
  - 展示比赛时间和 24 秒
  - 展示犯规、暂停、球权
  - 展示球员得分、犯规、上场状态
  - 点击标题可打开大屏设置抽屉
  - 大屏设置会保存到当前浏览器本地

- `auth.js`：登录与公共工具
  - 前端账号校验
  - 安全读取本地 JSON
  - HTML 转义
  - 程序跳转时放行关闭确认

- `unloadGuard.js`：关闭保护
  - 刷新、关闭标签页、关闭浏览器时弹出浏览器原生确认
  - 用于减少比赛现场误关页面风险

## 推荐现场使用方式

1. 电脑连接外接大屏。
2. 打开 `login.html` 登录。
3. 进入 `index.html` 配置赛事信息。
4. 点击“确定保存配置”。
5. 打开 `scoreCtrl.html`，放在电脑主屏幕操作。
6. 打开 `bigScreen.html`，拖到外接大屏并按 `F11` 全屏展示。

建议比赛现场使用同一台电脑、同一个浏览器打开控制台和大屏，这样同步最稳定。

## 快捷键

记分控制台支持：

- `空格`：开始/暂停比赛计时
- `A`：回表 24 秒
- `S`：回表 14 秒
- `D`：暂停 24 秒
- `F`：继续 24 秒

## 数据同步机制

项目使用浏览器本地能力同步数据：

- `localStorage`
  - `matchConfig`：赛事配置
  - `matchGameData`：比赛运行数据
  - `bigScreenDisplaySettings`：大屏显示设置
  - `basketballScoreAuth`：登录状态
  - `basketballScoreControlLock`：记分控制台主控锁

- `BroadcastChannel`
  - 同一浏览器多窗口之间实时同步比分数据

- `storage` 事件
  - 跨页面同步兜底

## GitHub Pages 部署

本项目是纯静态页面，可以直接部署到 GitHub Pages。

### 方式一：使用 GitHub Actions

仓库已包含：

```text
.github/workflows/pages.yml
```

推送到 `main` 分支后，GitHub Actions 会自动发布整个仓库根目录。

部署步骤：

1. 把项目推送到 GitHub 仓库。
2. 打开仓库 `Settings`。
3. 进入 `Pages`。
4. `Source` 选择 `GitHub Actions`。
5. 推送 `main` 分支后等待工作流完成。
6. 访问 GitHub Pages 提供的网址。

### 方式二：手动选择分支

也可以不用 Actions：

1. 打开仓库 `Settings`。
2. 进入 `Pages`。
3. `Source` 选择 `Deploy from a branch`。
4. 分支选择 `main`。
5. 目录选择 `/root`。
6. 保存后等待发布。

## GitHub Pages 访问入口

建议把 `login.html` 作为对外入口：

```text
https://你的用户名.github.io/仓库名/login.html
```

如果你希望访问根路径时自动进入登录页，可以后续把 `login.html` 作为首页，或增加一个跳转入口页。

## 商业化注意事项

当前登录是前端登录，账号密码写在 `auth.js` 中：

```js
const AUTH_USERS = [
  {
    username: 'demo',
    password: '123456',
    name: '试用账号',
  },
];
```

这种方式适合早期试用和简单访问控制，但不能防止懂技术的用户查看源码。正式商业化建议升级为后端登录、授权码签名或桌面离线授权。

## 当前限制

- 没有服务器，数据只保存在当前浏览器。
- 换电脑、换浏览器、清理浏览器数据后，比赛数据可能丢失。
- 主控锁只能防同一浏览器多开控制台，不能防不同电脑同时控制。
- 大屏设置只保存在当前浏览器。
